# Clean code VS Dirty code

## Qu'est-ce que du code sale ?

- Ne respectant pas une nomenclature définie.
- Ne détaillant pas son utilisation (nommage flou, documentation absente/insuffisante). 
- Ne suivant pas une architecture définie et évolutive (ex: sépération des responsabilités).

## Qu'est-ce que du code propre ?

- Flow explicite du projet (commits explicites, branches séparées et bien nommées).
- Facilement compréhensible et maintenable (agencement du projet et documentation/nommage).
- Respectant les principes de dev définis par l'équipe de dev(ex: KISS, SOLID, CUPID, ...)
